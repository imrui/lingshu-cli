> **"中枢一动，全栈皆通。"**
>
> **"One Core to Rule Them All."**

---

## 📜 灵枢宣言 (The Manifesto)

我们信仰 **Single Source of Truth** 的绝对力量。

在灵枢架构中，**`reference/` 不是文档，而是代码的灵魂**。它是所有微服务、所有前端组件、所有数据库以及所有 AI Agents 的共同祖先。

1.  **中枢神圣性**：`reference/docs/` 定义一切。代码只是文档的投影（Projection），而非反之。
2.  **绝对对齐**：任何不符合中枢定义的代码，皆为 bug。
3.  **AI 原生**：我们的架构天生为 AI 优化。人类负责在 `reference/` 中定义“意图”，AI 负责在此处将其“实现”。

## 📂 目录结构

- **`docs/`**: **真理之源**。存储 PRD、API 契约、状态机定义。
- **`experience/`**: **经验回环**。存储 Pitfalls（避坑指南）和 Best Practices。
- **`management/`**: **动态治理层**。下设 `plans/` (战略方案)、`tasks/` (战术任务)、`walkthroughs/` (存证存证) 与 `reports/` (审计报告)。
